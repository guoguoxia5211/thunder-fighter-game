## 📋 需求概述

用户在配置界面点击"保存"按钮后，将配置数据保存到配置系统（云端/服务器），而非本地备份。用户可以随时从配置系统调用已保存的配置。

## 🔧 技术实现建议

### 前端
```javascript
// 保存配置
async function saveConfig(configData) {
  const response = await fetch('/api/config/save', {
    method: 'POST',
    headers: { 'Content-Type': 'application/json' },
    body: JSON.stringify(configData)
  });
  return await response.json();
}

// 加载配置
async function loadConfig(configId) {
  const response = await fetch(`/api/config/load/${configId}`);
  return await response.json();
}
```

### 后端接口
- `POST /api/config/save` - 保存配置
- `GET /api/config/list` - 获取配置列表
- `GET /api/config/load/:id` - 加载指定配置
- `DELETE /api/config/:id` - 删除配置（可选）
- `GET /api/config/backup/list` - 获取 GitHub 备份历史

### 配置系统存储
- 数据库表：`user_configs`
- 字段：`id`, `user_id`, `config_name`, `config_data`, `created_at`, `updated_at`

### GitHub 备份实现
```javascript
// 备份配置到 GitHub
async function backupToGitHub(configData, userId) {
  const timestamp = new Date().toISOString().replace(/[:.]/g, '-');
  const fileName = `config_${userId}_${timestamp}.json`;
  const filePath = `backups/${fileName}`;
  
  // 1. 写入备份文件
  await fs.writeFile(filePath, JSON.stringify(configData, null, 2));
  
  // 2. Git 操作
  await exec('git add backups/');
  await exec(`git commit -m "backup: ${fileName}"`);
  await exec('git push origin main');
  
  // 3. 记录备份历史
  await db.backupHistory.create({
    fileName,
    filePath,
    userId,
    createdAt: new Date()
  });
}
```
