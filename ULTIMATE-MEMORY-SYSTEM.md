## 🔄 工作流程

### 会话开始
```
1. 读取 SESSION-STATE.md（必须）
2. 读取 MEMORY.md（主会话）
3. 读取 memory/YYYY-MM-DD.md（今天 + 昨天）
4. 运行 memory_search 召回相关记忆
5. 运行 memory_recall 向量检索
6. 注入上下文到当前会话
```

### 会话进行中
```
收到用户消息
    ↓
解析意图（纠正/决定/偏好/事实）
    ↓
WAL 协议：写 SESSION-STATE.md
    ↓
重要内容 → memory_store（向量存储）
    ↓
重大决策 → Git-Notes（知识图谱）
    ↓
回复用户
```

### 会话结束
```
1. 更新 SESSION-STATE.md 最终状态
2. 追加 memory/YYYY-MM-DD.md 日志
3. 记录变更到 CHANGELOG.md
4. 运行 backup-memory.sh（如有重大修改）
5. 每周：提炼到 MEMORY.md
```

## 🛡️ 保护协议（强制执行）

### 1. WAL 协议（Write-Ahead Log）

**规则：** 先写状态，再回复

**触发信号：**
| 信号 | 关键词 | 动作 |
|------|--------|------|
| 纠正 | "不对"/"应该是"/"实际上" | 写纠正内容 |
| 决定 | "用 X"/"选 Y"/"决定" | 写决定 |
| 偏好 | "我喜欢"/"总是"/"从不" | 写偏好 |
| 专有名词 | 人名/地名/公司名 | 写专有名词 |
| 具体值 | 数字/日期/ID/URL | 写具体值 |
| 担忧 | "担心"/"怕" | 写担忧 + 安抚 |

### 2. VBR 协议（Verify-Before-Reply）

**规则：** 修改项目前先验证 + 备份

**步骤：**
1. 读取当前文件
2. 创建 .bak 备份
3. 向用户确认
4. 执行修改
5. 验证功能
6. 汇报结果

### 3. 心跳检查

**频率：** 每 2-4 小时

**检查项：**
- [ ] SESSION-STATE.md 最新
- [ ] MEMORY.md 一致
- [ ] memory/ 日志完整
- [ ] 待处理动作无遗漏
- [ ] 项目状态同步

### 4. 确认协议

**重大改动前必须确认：**
- 删除文件/功能
- 重构核心代码
- 更换技术方案
- 修改用户偏好

## 🚨 紧急恢复

**当用户说"记忆不对"时：**

1. **立即停止** 当前任务
2. **读取** SESSION-STATE.md
3. **读取** MEMORY.md
4. **运行** memory_search
5. **对比** 用户说的 vs 记忆
6. **承认** 不一致（如有）
7. **询问** 正确信息
8. **更新** 所有记忆层
9. **记录** 事件到 memory/

**回应模板：**
```
你说得对，我立即检查记忆...

【SESSION-STATE.md】...
【MEMORY.md】...
【我的理解】...

哪里不一致？我立即修正并记录。
```
