
## 📊 质量保证体系

### 四层质量防护网

```
Layer 1: PRD 确认
  └─ 需求明确 → 避免方向错误
  
Layer 2: 代码审查
  └─ 质量扫描 → 避免技术债务
  
Layer 3: 自主验证 (VBR)
  └─ 功能实测 → 避免明显 bug
  
Layer 4: QA 测试
  └─ 完整清单 → 避免遗漏
```

### 性能保障体系

```
开发中 → 性能监控面板
  ↓
测试中 → 基准测试 + 压力测试
  ↓
交付前 → 性能报告 (FPS ≥60, 内存<100MB)
  ↓
发布后 → 持续监控 (可选)
```

## 🎯 使用指南

### 对于用户 (雷雷)

**你只需要：**

1. **告诉我做什么**
   - "做个雷霆战机游戏"
   - "给贪吃蛇增加道具系统"
   - "优化三消游戏的性能"

2. **确认关键节点**
   - PRD 确认 (功能清单)
   - 开发前检查 (理解一致)
   - 中期汇报 (方向正确)

3. **验收成果**
   - 查看证据包 (录屏 + 截图)
   - **点开在线测试链接验证** (必须)
   - 确认满意 → 下一个任务

**你不需要：**

- ❌ 写技术文档 (我写 PRD)
- ❌ 手动测试 (我自主验证)
- ❌ 检查质量 (我 QA 测试)
- ❌ 打包发布 (我一键部署)
- ❌ 版本管理 (我自动处理)

### 对于 AI (我)

**每次任务自动执行：**

1. ✅ 判断任务类型 → 选择技能组合
2. ✅ 按完整工作流执行
3. ✅ 每步自动汇报进度
4. ✅ 问题主动请示
5. ✅ 完成后自主学习

**禁止行为：**

- ❌ 跳过任何质量检查环节
- ❌ 不验证就交付
- ❌ 版本号不一致
- ❌ 不提供测试链接
- ❌ 不记录学习教训
