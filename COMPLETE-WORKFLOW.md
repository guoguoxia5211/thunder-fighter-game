## 🧪 QA 测试详细清单

### 功能完整性检查

```markdown
## 需求对照表

**用户需求：**
- [ ] 功能点 1
- [ ] 功能点 2
- [ ] 功能点 3

**实现状态：**
| 需求 | 状态 | 说明 |
|------|------|------|
| 功能 1 | ✅ | 完整实现 |
| 功能 2 | ✅ | 完整实现 |
| 功能 3 | ⚠️ | 简化 (需用户确认) |

**简化说明 (如有)：**
- 简化原因：
- 影响：
- 用户确认：[ ] 已确认
```

### 运行验证

```markdown
## 启动测试
- [ ] 页面正常加载
- [ ] 无 404 错误
- [ ] 所有资源加载完成

## 操作测试
- [ ] 玩家控制响应正常
- [ ] 键盘/触摸控制可用
- [ ] UI 按钮可点击

## 功能测试
- [ ] 核心玩法可运行
- [ ] 敌机/子弹系统正常
- [ ] 碰撞检测生效
- [ ] 计分系统正确
```

### BUG 扫描

```markdown
## 控制台检查
- [ ] 无 JavaScript 错误
- [ ] 无资源加载失败
- [ ] 无 CORS 错误

## 崩溃测试
- [ ] 长时间运行不崩溃
- [ ] 快速操作不崩溃
- [ ] 边界条件不崩溃

## 性能检查
- [ ] FPS ≥60 (平均)
- [ ] 内存稳定 (无泄漏)
- [ ] 无明显卡顿
```

## 🚨 禁止行为

### 开发阶段
- ❌ 擅自简化/删减功能
- ❌ 需求变更不请示

### QA 测试阶段
- ❌ 跳过 QA 测试
- ❌ 测试不通过就交付
- ❌ 测试报告造假
- ❌ 发现问题不记录

### 版本管理阶段
- ❌ 版本号不一致
- ❌ 不更新 VERSION.json
- ❌ 不创建 Git 标签

### 发布阶段
- ❌ 不同步 GitHub
- ❌ 不提供在线链接
- ❌ 跳过一致性检查
