## 版本管理规范

### 版本号格式
`v主版本。次版本.修订号`

- **主版本**：重大功能更新/架构变化
- **次版本**：新功能添加
- **修订号**：bug 修复/小优化

### 更新流程
1. 更新 VERSION.md 的版本号
2. 记录变更内容
3. git commit 标注版本号
4. git push 部署

### Commit 规范
```
v1.1.0: 添加可视化编辑器
v1.0.1: 修复 XX bug
v1.1.0: 添加 XX 功能
```
