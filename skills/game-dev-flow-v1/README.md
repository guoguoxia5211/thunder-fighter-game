# 游戏开发流程 1.0 - 快速参考

**完整文档：** SKILL.md

---

## 🎯 一句话总结

**每个游戏都内置美术替换系统，密码保护，实时预览。**

---

## 🔄 6 个阶段

```
1. 需求确认 → 2. 开发实现 → 3. QA 测试 → 4. 美术替换 → 5. 版本管理 → 6. 部署同步
```

---

## 🚨 绝对不能做的事

### 需求
- ❌ PRD 未确认就开工

### 开发
- ❌ 不做开发前检查
- ❌ 用 Canvas 绘制（必须用图片）

### 测试
- ❌ 让用户当测试员

### 美术
- ❌ 游戏不包含美术替换系统

### 版本
- ❌ 版本号不一致

### 部署
- ❌ 用 Python HTTP 服务（必须用 Nginx）

### 沟通
- ❌ 挤牙膏式开发

---

## ✅ 交付前必查

- [ ] 三轮 QA 测试通过
- [ ] 美术替换可用
- [ ] 版本号一致
- [ ] 链接能打开（自己先测试）
