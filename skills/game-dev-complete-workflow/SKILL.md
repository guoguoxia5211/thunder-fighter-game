# 游戏开发完整工作流 (Game Development Complete Workflow)

**版本：** 1.0  
**创建：** 2026-03-10  
**制定者：** 雷雷  
**优先级：** 🔥 P0 (所有游戏项目强制执行)  
**触发条件：** 用户下达任何游戏开发任务时自动执行

---

## 🎯 核心原则

### 三致原则
**GitHub 仓库版本号 = 服务器版本号 = 在线测试链接版本号**

**必须完全一致，不允许出现任何偏差。**

### 功能完整性原则
- ✅ 游戏必须可正常运行才交付
- ✅ 不擅自简化/删减任何功能
- ✅ 需求变更必须先请示用户
- ✅ 功能完整性逐项对照检查
- ✅ 强制运行验证 + 截图证据

### QA 测试原则
- ✅ 所有功能必须通过 QA 测试
- ✅ 测试不通过 → 返回修复
- ✅ 测试报告必须完整记录

### PRD 原则
- ✅ **需求不明确不开工**
- ✅ **PRD 文档必须用户确认**
- ✅ **开发对照 PRD 功能清单**
- ✅ **验收对照 PRD 验收标准**

### 工作流增强原则
- ✅ **每次交付附带测试证据包**
- ✅ **开工前做开发前检查**
- ✅ **进度 50% 时主动汇报**
- ✅ **每个版本写变更日志**

### 测试链接原则
- ✅ **必须提供在线测试链接**
- ✅ **用户必须点开验证**
- ✅ **证据包 + 实测，双重保障**
- ✅ **GitHub 版本 + 服务器版本都需提供**
- ✅ **两处版本号必须完全一致**

---

## 🔄 完整工作流 (11 步 + 4 增强)

```
0. 【PRD 创建】
   ├─ 创建游戏需求文档
   ├─ 列出功能清单 + 验收标准
   └─ 用户确认 → 定稿
      ↓
【增强 1】开发前检查 → 用户确认
      ↓
1. 需求分析 → 基于 PRD
      ↓
2. 执行开发 → 对照 PRD
      ↓
【增强 2】中期进度汇报 (50% 时) → 用户确认
      ↓
3. 【QA 测试】
   ├─ 功能完整性 (对照 PRD)
   ├─ 运行验证 (能否正常启动)
   ├─ 操作测试 (控制是否响应)
   ├─ BUG 扫描 (控制台/崩溃)
   └─ 性能测试 (FPS ≥60)
      ↓
【增强 3】生成测试证据包
   ├─ 操作录屏 (30 秒)
   ├─ 功能截图 (6-10 张)
   ├─ 性能报告 (FPS/内存)
   └─ 测试清单 (全部打勾)
      ↓
4. QA 通过？
   ├─ 否 → 返回步骤 2 修复
   └─ 是 → 继续
      ↓
5. 版本管理 → 升级版本号 (V1.1.1)
      ↓
【增强 4】生成版本变更日志
      ↓
6. GitHub 同步 → 推送代码 + 标签
      ↓
7. 双渠道部署
   ├─ GitHub Pages (代码仓库)
   └─ 服务器部署 (在线测试)
      ↓
8. 一致性检查 → GitHub=服务器=测试链接
      ↓
8. 一致性检查 → GitHub=服务器=测试链接
      ↓
9. 汇报完成 → 附完整报告 + 证据包 + 测试链接
```

## 📋 详细执行步骤

### 步骤 0: PRD 创建

**触发：** 用户下达游戏任务

**执行：**
1. 创建 PRD 文档
2. 列出功能清单 (P0/P1/P2 优先级)
3. 制定验收标准
4. 等待用户确认

**模板：**
```markdown
# 游戏需求文档 (PRD)

**项目：** [游戏名称]
**版本：** V1.0.0

## 功能清单
| ID | 功能 | 优先级 | 验收标准 |
|----|------|--------|----------|
| F01 | 玩家控制 | P0 | 响应<100ms |
| F02 | 发射子弹 | P0 | 冷却 0.3 秒 |

## 用户确认
- [ ] 功能清单正确
- [ ] 可以开始开发
```

**禁止：** PRD 未确认就开工

---

### 增强 1: 开发前检查

**触发：** PRD 确认后

**执行：**
1. 检查 PRD 已创建
2. 检查 PRD 已用户确认
3. 检查功能清单理解无误
4. 检查验收标准明确
5. 用户确认理解一致

**模板：**
```markdown
## 开发前检查

- [x] PRD 已创建
- [x] PRD 已用户确认
- [x] 功能清单理解无误
- [x] 验收标准明确

**我确认：理解一致，可以开工。**
```

**禁止：** 不做开发前检查就开工

---

### 步骤 1-2: 需求分析 + 执行开发

**触发：** 开发前检查确认

**执行：**
1. 基于 PRD 分析需求
2. 对照 PRD 功能清单开发
3. 不擅自简化/删减
4. 需求变更必须请示

**禁止：** 擅自变更需求

---

### 增强 2: 中期进度汇报

**触发：** 开发进度达到 50%

**执行：**
1. 汇报已完成功能
2. 汇报进行中功能
3. 汇报遇到的问题
4. 提供进度截图
5. 等待用户确认方向

**模板：**
```markdown
## 中期进度汇报

**进度：** 50%

### ✅ 已完成
- [x] F01 - 玩家控制

### 🔄 进行中
- [ ] F02 - 发射子弹 (50%)

### ⚠️ 问题
- 问题 1：...
- 建议：...

**请确认：方向是否正确？**
```

**禁止：** 进度 50% 时不汇报

---

### 步骤 3: QA 测试

**触发：** 开发完成

**执行：**
1. 功能完整性检查 (对照 PRD)
2. 运行验证 (能否正常启动)
3. 操作测试 (控制是否响应)
4. BUG 扫描 (控制台/崩溃)
5. 性能测试 (FPS ≥60)

**检查清单：**
```markdown
## QA 测试清单

- [ ] 游戏能启动
- [ ] 控制有响应
- [ ] 所有功能可测试
- [ ] 玩 1 分钟不崩溃
- [ ] 控制台无报错
- [ ] FPS ≥60
```

**禁止：** QA 测试不通过就交付

---

### 增强 3: 生成测试证据包

**触发：** QA 测试完成

**执行：**
1. 录制操作视频 (30 秒)
2. 截取功能截图 (6-10 张)
3. 生成性能报告 (FPS/内存)
4. 填写测试清单 (全部打勾)

**模板：**
```markdown
## 测试证据包

### 📹 操作录屏
- 文件：`/evidence/demo.mp4`
- 时长：30 秒

### 📸 功能截图
- 01-main-menu.png
- 02-player-control.png
- 03-shooting.png

### 📊 性能报告
- FPS: 58 (平均)
- 内存：78MB

### ✅ 测试清单
- [x] 游戏能启动
- [x] 控制有响应
- [x] 无控制台报错

**结论：测试通过，可以直接玩 ✅**
```

**禁止：** 不生成证据包就交付

---

### 步骤 4-5: QA 通过 + 版本管理

**触发：** QA 测试通过

**执行：**
1. 判断版本类型 (major/minor/patch)
2. 升级版本号 (V1.1.1)
3. 更新 VERSION.json
4. 创建 Git 标签

**版本规则：**
- V1.x.x - 重大更新
- Vx.1.x - 功能增加
- Vx.x.1 - BUG 修复

**禁止：** 版本号不一致

---

### 增强 4: 生成版本变更日志

**触发：** 版本管理完成

**执行：**
1. 列出新增功能
2. 列出修复 BUG
3. 列出性能优化
4. 数据对比 (与上一版本)

**模板：**
```markdown
# 版本变更日志

## V1.0.1 (2026-03-10)

### 新增
- ✅ F01 - 玩家控制
- ✅ F02 - 发射子弹

### 修复
- 🐛 修复子弹速度过慢

### 数据对比
| 指标 | V1.0.0 | V1.0.1 | 变化 |
|------|--------|--------|------|
| 功能数 | 3 | 6 | +3 ⬆️ |
| FPS | 45 | 58 | +13 ⬆️ |
```

**禁止：** 不写版本日志

---

### 步骤 6-8: GitHub 同步 + 在线链接 + 一致性检查

**触发：** 版本管理完成

**执行：**
1. 推送代码到 GitHub
2. 推送 Git 标签
3. 生成在线测试链接
4. 执行一致性检查

**一致性检查：**
```markdown
## 一致性检查

| 位置 | 版本号 | 状态 |
|------|--------|------|
| 本地 VERSION.json | V1.0.1 | ✅ |
| Git 标签 | V1.0.1 | ✅ |
| GitHub 远程 | V1.0.1 | ✅ |
| 服务器 | V1.0.1 | ✅ |
| 测试链接 | V1.0.1 | ✅ |

**结论：三处版本号完全一致 ✅**
```

**禁止：** 版本号不一致、不同步 GitHub、不提供测试链接

---
