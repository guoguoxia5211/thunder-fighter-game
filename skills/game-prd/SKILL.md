# 游戏需求文档技能 (Game PRD)

**版本：** 1.0  
**创建：** 2026-03-10  
**目标：** 需求文档化，防止理解偏差和返工

---

## 🎯 核心原则

**需求不明确不开工。**

每次游戏任务开始前，必须创建 PRD 文档，用户确认后才执行。

---

## 📋 PRD 文档模板

```markdown
# 游戏需求文档 (PRD)

**项目名称：** [游戏名称]
**版本：** V1.0.0
**创建时间：** 2026-03-10
**最后更新：** 2026-03-10

---

## 1. 游戏概述

### 1.1 游戏类型
- [ ] 射击游戏 (STG)
- [ ] 益智游戏 (Puzzle)
- [ ] 动作游戏 (ACT)
- [ ] 其他：____

### 1.2 核心玩法
[用 1-2 句话描述游戏核心玩法]

### 1.3 目标用户
- 主要用户：[描述]
- 使用场景：[移动端/桌面/两者]

## 2. 功能清单

### 2.1 功能优先级说明
- **P0** - 核心功能，没有就不是这个游戏
- **P1** - 重要功能，影响游戏体验
- **P2** - 锦上添花，可后续迭代

### 2.2 功能列表

| ID | 功能名称 | 优先级 | 功能描述 | 验收标准 |
|----|----------|--------|----------|----------|
| F01 | 玩家控制 | P0 | 描述如何控制玩家 | 响应<100ms，无延迟 |
| F02 | 敌机系统 | P0 | 描述敌机行为 | 3 种敌机，不同属性 |
| F03 | 子弹系统 | P0 | 描述子弹发射 | 按空格发射，有冷却 |
| F04 | 碰撞检测 | P0 | 描述碰撞逻辑 | 击中掉血，误差<5px |
| F05 | 计分系统 | P1 | 描述计分规则 | 击毁敌机 +100 分 |
| ... | ... | ... | ... | ... |

---

### 3.3 包体大小
- 目标：<10MB

- [ ] FPS ≥60

### 5.3 交付物

- [ ] 在线测试链接
- [ ] 版本号 (V1.0.0)
- [ ] 版本一致性检查通过
```

## 🔧 需求确认流程

```
1. 用户下达游戏任务 (口头/文字)
      ↓
2. 我创建 PRD 文档草稿
      ↓
3. 列出功能清单 + 验收标准
      ↓
4. 用户确认/修改
      ↓
5. 双方确认 → PRD 定稿
      ↓
6. 开始开发
```

---

## 📝 PRD 创建示例

### 用户说：
> "做个雷霆战机，能发射子弹打敌机"

### 我创建的 PRD：

```markdown
# 雷霆战机 PRD

## 功能清单

| ID | 功能 | 优先级 | 描述 | 验收标准 |
|----|------|--------|------|----------|
| F01 | 玩家控制 | P0 | 键盘左右移动，触摸拖动 | 响应<100ms |
| F02 | 发射子弹 | P0 | 按空格发射，自动连发 | 冷却 0.3 秒 |
| F03 | 敌机生成 | P0 | 从顶部随机生成 | 每秒 1-2 架 |
| F04 | 敌机移动 | P0 | 向下移动，速度不同 | 3 种速度 |
| F05 | 碰撞检测 | P0 | 子弹击中敌机 | 敌机消失 + 计分 |
| F06 | 计分系统 | P1 | 击毁敌机 +100 分 | 显示在左上角 |

## 用户确认
- [ ] 功能清单正确
- [ ] 验收标准合理
- [ ] 可以开始开发
```

---

## 🚨 禁止行为

- ❌ 没有 PRD 就开工
- ❌ PRD 未经用户确认
- ❌ 开发过程中擅自变更需求
- ❌ 需求变更不更新 PRD

---

## 🔄 需求变更流程

```
发现需要变更需求
      ↓
更新 PRD 文档 (变更记录)
      ↓
用户确认变更
      ↓
继续开发
```

---

## 📋 与工作流整合

```
0. 创建 PRD → 用户确认
      ↓
1. 需求分析 (基于 PRD)
      ↓
2. 执行开发 (对照 PRD 功能清单)
      ↓
3. QA 测试 (对照 PRD 验收标准)
      ↓
4-9. 后续流程...
```

---

*此技能在所有游戏项目任务中强制执行，PRD 未确认不开工*
