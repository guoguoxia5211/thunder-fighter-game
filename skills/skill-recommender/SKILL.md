# Skill Recommender - 智能技能推荐系统

## 核心功能
分析用户任务，自动推荐最适合的技能组合，提升任务完成质量和效率。

## 触发条件
每次收到用户任务/请求时自动执行。

## 工作流程

### 1. 任务分析
```
收到任务 → 解构需求 → 识别关键动作 → 匹配技能类型
```

### 2. 技能匹配
根据任务类型匹配对应技能：

| 任务类型 | 推荐技能 | 触发关键词 |
|---------|---------|-----------|
| **游戏开发** | game-dev-master, game-developer, game-engine | 游戏、玩法、Unity、Three.js、碰撞、物理 |
| **UI 设计** | frontend-design, happy-hues | 界面、UI、颜色、布局、美观 |
| **3D 建模** | 3d-modeler, blender | 模型、3D、Blender、 Maya、角色、场景 |
| **动画** | animator, animations | 动画、关键帧、动作、运动 |
| **美术** | master-artist, character-designer, environment-artist | 美术、角色、场景、原画、概念 |
| **文档** | docs-cog, codebase-documenter | 文档、README、说明、教程 |
| **调试** | bug-investigation | bug、错误、问题、修复 |
| **性能** | game-optimization-master, performance-profiler | 优化、性能、帧率、内存 |
| **多人游戏** | multiplayer-networking | 多人、联机、同步、网络 |
| **天气查询** | weather | 天气、气温、预报 |
| **技能查找** | find-skills, clawhub | 技能、安装、扩展 |
| **GitHub** | github, gh-issues | PR、issue、仓库、提交 |
| **浏览器自动化** | openclaw-browser-quality | 网页、截图、自动化、交互 |
| **钉钉提醒** | dingtalk-cron-job | 提醒、定时、通知 |
| **文件发送** | dingtalk-send-media | 发送、图片、文件 |
| **深度思考** | deep-thinking | 复杂、架构、决策、方案 |
| **第一性原理** | first-principles-decomposer | 本质、原理、从 0、基础 |

### 3. 推荐输出格式

```
🎯 **技能推荐**

**主要技能：**
- 🎮 game-dev-master - 游戏开发核心逻辑

**辅助技能：**
- 🎨 frontend-design - UI 界面美化
- 🐛 bug-investigation - 调试支持

**推荐理由：**
你的任务需要游戏开发 + UI 设计，建议使用以上技能组合。

**是否启用？** 回复"是"或指定技能名称。
```

### 4. 用户响应处理

- **用户同意** → 激活推荐技能，按技能规范执行
- **用户指定** → 使用用户选择的技能
- **用户跳过** → 使用默认能力执行

## 技能激活规则

1. **默认不主动调用** - 先推荐，等用户确认
2. **紧急任务** - 可直接调用最相关技能
3. **技能冲突** - 选择优先级最高的
4. **多技能协作** - 明确分工，避免重复

## 优先级

1. 用户明确指定的技能
2. 系统推荐且用户确认的技能
3. 根据上下文自动选择的技能
4. 默认通用能力

## 示例

### 示例 1：游戏开发任务
**用户：** "做个贪吃蛇游戏"

**推荐：**
```
🎯 **技能推荐**

**主要技能：**
- 🎮 game-dev-master - 游戏开发专家系统（强制代码审查 + 自主验证）
- 🎲 game-engine - HTML5 游戏引擎

**辅助技能：**
- 🎨 frontend-design - 界面美化
- 🧪 e2e-testing-patterns - 测试覆盖

**推荐理由：**
贪吃蛇需要游戏核心逻辑 + 渲染引擎，game-dev-master 确保代码质量。

**是否启用？** 回复"是"或指定技能名称。
```

### 示例 2：UI 设计任务
**用户：** "设计个开始界面"

**推荐：**
```
🎯 **技能推荐**

**主要技能：**
- 🎨 frontend-design - 专家级 UI 设计

**辅助技能：**
- 🌈 happy-hues - 配色方案推荐

**推荐理由：**
界面设计需要专业 UI 能力 + 配色建议。

**是否启用？** 回复"是"或指定技能名称。
```

### 示例 3：调试任务
**用户：** "游戏卡顿，帧率只有 20"

**推荐：**
```
🎯 **技能推荐**

**主要技能：**
- ⚡ game-optimization-master - 性能优化专家
- 🔍 performance-profiler - 性能分析

**辅助技能：**
- 🐛 bug-investigation - 问题排查

**推荐理由：**
性能问题需要专业优化 + 性能分析工具。

**是否启用？** 回复"是"或指定技能名称。
```

## 注意事项

1. **不要过度推荐** - 简单任务不需要技能
2. **技能数量** - 每次推荐 1-3 个，最多 5 个
3. **说明清晰** - 让用户知道每个技能的作用
4. **尊重选择** - 用户可以拒绝或修改

## 技能库更新

定期查看可用技能列表，更新匹配规则：
- 新技能安装 → 添加匹配规则
- 技能失效 → 移除推荐
- 用户反馈 → 优化推荐算法

---

**目标：** 让每次任务都用上最合适的工具，提升效率和质量。
