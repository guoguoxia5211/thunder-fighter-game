# VBR 自主验证技能 (Autonomous Verification)

**版本：** 1.0  
**创建：** 2026-03-10  
**目标：** 代码存在 ≠ 功能可用，必须自主验证

---

## 🎯 核心原则

**绝不让用户当测试员。**

写完代码后，必须主动验证功能是否真正可用，而不是只检查语法。

---

## 📋 验证清单

### 前端项目 (HTML/JS 游戏)

| 检查项 | 方法 | 工具 |
|--------|------|------|
| 语法正确 | 解析 HTML/JS | `node --check` / 浏览器控制台 |
| 资源加载 | 检查文件路径 | `ls -la` 验证路径存在 |
| 功能可用 | 浏览器打开测试 | `browser` 工具截图/交互 |
| 移动端适配 | 响应式检查 | 浏览器模拟器 |
| 控制台无报错 | 检查 JS 错误 | 浏览器控制台日志 |

### 后端/脚本项目

| 检查项 | 方法 | 工具 |
|--------|------|------|
| 语法正确 | 解析执行 | `node --check file.js` |
| 依赖完整 | 检查 imports | `npm ls` / 手动检查 |
| 功能测试 | 运行测试用例 | 自动执行测试脚本 |
| 边界条件 | 极端值测试 | 自动构造测试数据 |

---

## 🔧 验证流程

```
1. 代码完成 → 写 VBR 检查清单
2. 语法检查 → 确保无错误
3. 资源验证 → 确保文件存在
4. 功能测试 → 浏览器/命令行实测
5. 截图记录 → 保存验证证据
6. 问题修复 → 发现问题立即修
7. 最终确认 → 再次验证通过
8. 汇报结果 → 告诉用户验证完成
```

---

## 📝 输出格式

验证完成后，汇报格式：

```markdown
## ✅ VBR 验证完成

**验证项：** [项目名称]
**验证时间：** YYYY-MM-DD HH:mm

### 检查结果
- [x] 语法检查通过
- [x] 资源加载正常
- [x] 功能测试通过
- [x] 控制台无报错
- [x] 移动端适配正常

### 验证截图
[DING:IMAGE path="/root/.openclaw/workspace/vbr-screenshots/xxx.png"]

### 发现的问题 (如有)
- 问题 1 → 已修复
- 问题 2 → 已修复

### 结论
功能可用，可以交付用户使用。
```

---

## 🚨 禁止行为

- ❌ 只检查语法不测试功能
- ❌ 让用户发现明显 bug
- ❌ 说"应该没问题"但不验证
- ❌ 跳过截图记录环节

---

## 🧠 自主学习

每次验证后记录：
1. 发现了什么问题？
2. 如何修复的？
3. 下次如何避免？

积累到 3 次相似问题 → 更新为全局规则。
