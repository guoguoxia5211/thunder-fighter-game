# 自主任务发现技能 (Autonomous Task Discovery)

**版本：** 1.0  
**创建：** 2026-03-10  
**目标：** 主动发现待办事项，减少等待用户指示

---

## 🎯 核心原则

**不被动等待，主动发现问题和机会。**

---

## 📋 检查清单

### 每日自动检查 (Heartbeat 触发)

| 检查项 | 频率 | 动作 |
|--------|------|------|
| Git 状态 | 每次会话 | `git status` 检查未提交变更 |
| 项目进度 | 每日 | 读取 PROJECTS.md 对比实际状态 |
| 待办事项 | 每日 | 检查 SESSION-STATE.md 过期任务 |
| 代码质量 | 每周 | 扫描常见 bug 模式 |
| 依赖更新 | 每周 | 检查 npm/系统包更新 |

### 项目状态检查

```bash
# 检查每个项目目录
for project in snake-ultimate thunder-fighter; do
  cd /root/.openclaw/workspace/$project
  git status --short
  # 发现未提交变更 → 记录待办
done
```

### 代码质量扫描

检查模式：
- `console.log` 调试代码残留
- 未使用的变量/函数
- 明显的逻辑错误
- TODO/FIXME 注释

---

## 🔧 发现任务后的处理流程

```
1. 发现问题 → 记录到 SESSION-STATE.md
2. 评估优先级 → 高/中/低
3. 高优先级 → 立即处理
4. 中优先级 → 加入待办，下次处理
5. 低优先级 → 记录到 HEARTBEAT.md
6. 汇报 → 告诉用户发现了什么 + 已做什么
```

---

## 📝 汇报格式

```markdown
## 🔍 自主检查发现

**检查时间：** YYYY-MM-DD HH:mm
**检查范围：** [项目名称/范围]

### 发现的问题
1. **问题描述** → 优先级：高
   - 影响：...
   - 建议：...
   - 状态：已修复/待处理

### 已执行的操作
- [x] 操作 1
- [x] 操作 2

### 待确认事项
- [ ] 需要用户决策的事项

### 下次检查
计划：[时间/触发条件]
```

---

## 🚨 边界

### 可以自主执行的
- ✅ 代码格式修复
- ✅ 调试代码清理
- ✅ 文档更新
- ✅ 简单 bug 修复
- ✅ Git 提交（有明确变更时）

### 需要请示的
- ❌ 架构变更
- ❌ 破坏性修改
- ❌ 外部 API 调用
- ❌ 删除重要文件

---

## 🧠 学习规则

每次任务发现后记录：
1. 是如何发现的？
2. 用户是否认可这个任务？
3. 下次是否继续检查这类问题？
