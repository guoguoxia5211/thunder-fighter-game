# 📋 游戏开发工作流规范

**制定日期：** 2026-03-10  
**制定者：** 雷雷  
**优先级：** 🔥 P0 (所有游戏项目强制执行)

---

## 🎯 核心原则

### 三致原则
**GitHub 仓库版本号 = 服务器版本号 = 在线测试链接版本号**

**必须完全一致，不允许出现任何偏差。**

---

## 🏷️ 版本号规则

### 格式
```
V[大版本].[功能版本].[BUG 版本]
```

### 升级规则

| 变更类型 | 升级位置 | 示例 |
|----------|----------|------|
| **重大更新** | 第一位 | 架构变更/核心玩法 | V1.0.0 → V2.0.0 |
| **功能增加** | 第二位 | 新系统/新功能 | V1.1.0 → V1.2.0 |
| **BUG 修复** | 第三位 | 优化/修复 | V1.1.1 → V1.1.2 |

---

## 🔄 标准工作流

```
用户下达游戏任务
      ↓
执行开发任务
      ↓
自主验证 (VBR)
      ↓
性能测试 (FPS ≥60)
      ↓
代码审查 (无 Blocker)
      ↓
【版本管理】← 必须执行
      ↓
【GitHub 同步】← 必须执行
      ↓
【在线测试链接】← 必须执行
      ↓
版本号一致性检查 ← 必须通过
      ↓
汇报完成
```

```markdown
## ✅ 游戏任务完成

**项目：** [项目名称]

**时间：** 2026-03-10 HH:mm

### 📦 版本管理
- [x] VERSION.json 更新 (V1.2.3)
- [x] Git 标签创建 (V1.2.3)
- [x] GitHub 推送完成

### 🌐 在线测试
- **主链接：** https://[domain]/[project]/
- **版本链接：** https://[domain]/[project]/versions/V1.2.3/

### ✅ 版本号一致性
| 位置 | 版本号 | 状态 |
|------|--------|------|
| 本地 VERSION.json | V1.2.3 | ✅ |
| Git 标签 | V1.2.3 | ✅ |
| GitHub 远程 | V1.2.3 | ✅ |
| 服务器 | V1.2.3 | ✅ |
| 测试链接 | V1.2.3 | ✅ |

**结论：** 三处版本号完全一致 ✅
```

---

## 🚨 禁止行为

- ❌ 版本号不一致
- ❌ 更新后不做版本管理
- ❌ 不同步 GitHub
- ❌ 不提供在线测试链接
- ❌ 跳过一致性检查

---

## 📋 快速命令

```bash
# 版本升级
./version-bump.sh minor  # 或 major/patch

# GitHub 同步
git push origin main && git push origin V1.2.3

# 一致性检查
cat VERSION.json | jq -r '.version'
git describe --tags --abbrev=0
```
